# Docker Release Checker

Автоматизированная система мониторинга и скачивания новых релизов Docker Desktop для Windows.

## 📋 Описание

Docker Release Checker - это веб-приложение на Python, которое автоматически:
- Мониторит страницу релизов Docker Desktop
- Определяет доступность новых версий
- Скачивает новые релизы для Windows
- Сохраняет файлы с версионированными именами
- Сравнивает версии и избегает повторных скачиваний

Приложение работает как сервис в Docker контейнере и регулярно проверяет обновления согласно настроенному интервалу.

## 🚀 Возможности

- ✅ **Гибкая конфигурация**: все параметры (URL, интервал, селекторы) настраиваются через JSON
- ✅ **Динамическая перезагрузка**: конфигурация перечитывается каждый цикл без перезапуска контейнера
- ✅ **Версионирование**: автоматическое сравнение версий с использованием библиотеки `packaging`
- ✅ **CSS селекторы**: гибкий поиск ссылок на релизы через BeautifulSoup
- ✅ **Regex паттерны**: извлечение номеров версий по настраиваемым шаблонам
- ✅ **Логирование**: подробные логи в консоль (stdout) для мониторинга
- ✅ **Docker volumes**: скачанные файлы сохраняются на хосте
- ✅ **Прогресс скачивания**: отображение процента и размера при загрузке больших файлов
- ✅ **Устойчивость к ошибкам**: автоматическое восстановление после сбоев

## ⚙️ Конфигурация

> ⚠️ **ВАЖНО**: Дефолтная конфигурация требует настройки! Страница Docker release notes не содержит прямых ссылок на скачивание. См. [CONFIG_EXAMPLES.md](CONFIG_EXAMPLES.md) для примеров правильной настройки.

Файл `config.json` содержит все настройки приложения:

```json
{
  "check_interval_seconds": 3600,
  "release_page_url": "https://docs.docker.com/desktop/release-notes/",
  "css_selector_link": "a[href*='windows']",
  "css_selector_version": "h2",
  "version_pattern": "(\\d+\\.\\d+\\.\\d+)",
  "download_path": "/downloads",
  "filename_template": "DockerDesktop-{version}-windows.exe",
  "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
```

### Параметры конфигурации

| Параметр | Тип | Описание | Пример |
|----------|-----|----------|--------|
| `check_interval_seconds` | int | Интервал проверки обновлений в секундах | `3600` (1 час) |
| `release_page_url` | string | URL страницы с релизами | `https://docs.docker.com/desktop/release-notes/` |
| `css_selector_link` | string | CSS селектор для поиска ссылки на скачивание | `a[href*='windows']` |
| `css_selector_version` | string | CSS селектор для поиска версии (опционально) | `h2` |
| `version_pattern` | string | Regex для извлечения номера версии | `(\\d+\\.\\d+\\.\\d+)` |
| `download_path` | string | Путь сохранения файлов внутри контейнера | `/downloads` |
| `filename_template` | string | Шаблон имени файла с `{version}` | `DockerDesktop-{version}-windows.exe` |
| `user_agent` | string | User-Agent для HTTP запросов | `Mozilla/5.0...` |

### Примеры CSS селекторов

```json
// Поиск всех ссылок, содержащих "windows" в href
"css_selector_link": "a[href*='windows']"

// Поиск ссылки по классу
"css_selector_link": "a.download-link"

// Поиск ссылки внутри конкретного блока
"css_selector_link": "div.release-section a[href$='.exe']"

// Поиск первого заголовка второго уровня
"css_selector_version": "h2:first-of-type"
```

### Примеры regex паттернов

```json
// Версия формата X.Y.Z
"version_pattern": "(\\d+\\.\\d+\\.\\d+)"

// Версия с дополнительной цифрой X.Y.Z.W
"version_pattern": "(\\d+\\.\\d+\\.\\d+\\.\\d+)"

// Версия с возможным префиксом "v"
"version_pattern": "v?(\\d+\\.\\d+\\.\\d+)"
```

## 🛠️ Установка и запуск

### Вариант 1: Запуск в Docker (рекомендуется)

1. **Убедитесь, что Docker и Docker Compose установлены**

2. **Клонируйте репозиторий или скопируйте файлы проекта**

3. **Настройте конфигурацию** в файле `config.json` под ваши нужды

4. **Соберите и запустите контейнер:**

```bash
docker-compose up --build
```

Для запуска в фоновом режиме (как сервис):

```bash
docker-compose up -d
```

5. **Просмотр логов:**

```bash
docker-compose logs -f
```

6. **Остановка сервиса:**

```bash
docker-compose down
```

### Вариант 1.1: Запуск без docker-compose (через .cmd)

В проекте есть готовые скрипты для Windows:

```bat
build.cmd
start.cmd
```

`build.cmd` выполняет сборку образа:

```bat
docker build -t ghcr.io/infarh/docker-release-checker:latest .
```

`start.cmd` запускает контейнер с параметрами:
- директория для скачивания релизов на хосте: `D:\!Downloads\Docker`
- конфигурация на хосте: `D:\!Downloads\Docker\config.json`
- проброс volume для релизов: `-v "D:\!Downloads\Docker:/downloads"`
- проброс volume для конфигурации: `-v "D:\!Downloads\Docker\config.json:/app/config.json:ro"`
- политика перезапуска: `--restart unless-stopped`

Эквивалент прямого запуска `docker run`:

```bat
docker run -d ^
  --name docker-release-checker ^
  --restart unless-stopped ^
  -v "D:\!Downloads\Docker:/downloads" ^
  -v "D:\!Downloads\Docker\config.json:/app/config.json:ro" ^
  ghcr.io/infarh/docker-release-checker:latest
```

Просмотр логов:

```bat
docker logs -f docker-release-checker
```

### Вариант 2: Локальный запуск (для разработки)

1. **Создайте виртуальное окружение:**

```bash
python -m venv venv
```

2. **Активируйте виртуальное окружение:**

Windows:
```bash
venv\Scripts\activate
```

Linux/Mac:
```bash
source venv/bin/activate
```

3. **Установите зависимости:**

```bash
pip install -r requirements.txt
```

4. **Настройте `config.json`** (измените `download_path` на `./downloads`)

5. **Запустите приложение:**

```bash
python -m src.main
```

Для остановки нажмите `Ctrl+C`.

## 🐛 Решение проблем

### Проблема: Не находится ссылка на релиз

**Решение:** Проверьте и настройте `css_selector_link`:

1. Откройте страницу релизов в браузере
2. Используйте инструменты разработчика (F12)
3. Найдите элемент со ссылкой на скачивание
4. Скопируйте CSS селектор или создайте подходящий
5. Обновите `config.json`

### Проблема: Не извлекается версия

**Решение:** Настройте `version_pattern`:

1. Посмотрите на формат версии на странице (например, `4.28.0`, `v4.28.0`)
2. Создайте соответствующий regex паттерн
3. Протестируйте паттерн на сайте regex101.com
4. Обновите `config.json`

### Проблема: Контейнер постоянно перезапускается

**Решение:** Проверьте логи:

```bash
docker-compose logs
```

Частые причины:
- Ошибка в `config.json` (невалидный JSON)
- Недоступна страница релизов
- Проблемы с правами доступа к директории `downloads/`

### Проблема: Файлы не появляются в `./downloads/`

**Решение:**

1. Проверьте, что директория `downloads/` существует на хосте
2. Проверьте права доступа к директории
3. Убедитесь, что в `docker-compose.yml` правильно настроен volume mapping
